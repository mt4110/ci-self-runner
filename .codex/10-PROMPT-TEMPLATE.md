# Codex プロンプト雛形（負荷をかけず、自由にさせない）

あなたはこのリポジトリの実装補助。最初に以下を読み、従うこと:
- .codex/00-RULES-READ-FIRST.md
- docs/ci/*.md（特に SYSTEM / FLOW / RUNNER_ISOLATION / COLIMA_TUNING / RUNBOOK）
実装開始前に「変更対象ファイル一覧」と「最小差分の計画」を先に提示すること。

制約:
- リポ名変更禁止、`codex/` ディレクトリ禁止
- 新規導入は最小限（k8s/k9s/nix 不要）
- Docker Desktop は使わず colima を前提にする
- 大きい処理は分割（重いと察知したら分割/分散）
- 既存ファイルの全面置換や大規模リファクタはしない（差分最小）

進め方:
1) まず plan: 3〜7ステップの手順（各ステップに DONE 条件）
2) 変更するファイルのパスを列挙（追加/更新）
3) 実装は“小さく刻む”。1ステップ=1コミット相当の粒度
4) 各ステップで検証コマンドを提示（実行結果はユーザーが貼る）

出力形式:
- 先に Plan
- 次に 変更ファイル一覧
- 次に 差分（必要箇所だけ）
